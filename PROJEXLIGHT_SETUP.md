## 🐧 Linux Setup (Required Before First Run)

After downloading and extracting the CLI export on Linux/Ubuntu:

### 1. Make Scripts Executable

```bash
cd mcp-server
chmod +x *.sh
chmod +x templates/*.sh
```

### 2. Install Docker Compose V2 (Recommended)

The older Python-based `docker-compose` (v1) may have library conflicts. Use Docker Compose V2:

```bash
# Install Docker Compose plugin (V2)
sudo apt-get update
sudo apt-get install docker-compose-plugin

# Verify installation
docker compose version
```

> **Note:** If you see `URLSchemeUnknown: Not supported URL scheme http+docker` error, this means you're using the old `docker-compose` v1 with Python library conflicts. Install Docker Compose V2 as shown above.

## Configure MCP Server (Optional)

If your AI tool supports native MCP, configure it for better integration.

**Two connection methods:**
- **HTTP:** Connect to `http://localhost:8766` (requires MCP server running in Docker)
- **STDIO Bridge:** Run `node mcp-server/mcp-bridge.js` (local Node.js process)

### GitHub Copilot CLI
1. Run `copilot mcp` or press `Ctrl+M`
2. Select "Add New MCP Server"
3. Configure:

**Option A - HTTP:**
- **Name:** projexlight
- **Type:** HTTP (option 3)
- **URL:** http://localhost:8766

**Option B - STDIO Bridge:**
- **Name:** projexlight
- **Type:** STDIO (option 2)
- **Command:** node mcp-server/mcp-bridge.js

### Goose
Edit `~/.config/goose/config.yaml`:

**Option A - HTTP:**
```yaml
mcp_servers:
  projexlight:
    uri: http://localhost:8766
```

**Option B - STDIO Bridge:**
```yaml
mcp_servers:
  projexlight:
    command: node
    args: ["mcp-server/mcp-bridge.js"]
```

### VS Code (Cline Extension)

## Troubleshooting

**MCP Server not working?**
```bash
curl http://localhost:8766/health
./setup-dev-mcp.sh logs
cd mcp-server && ./setup-all.sh --force
```

### Linux-Specific Issues

**Permission denied when running scripts?**
```bash
cd mcp-server
chmod +x *.sh
chmod +x templates/*.sh
```

**`URLSchemeUnknown: Not supported URL scheme http+docker` error?**

This is a Python library conflict with the old `docker-compose` v1. Fix by installing Docker Compose V2:
```bash
sudo apt-get update
sudo apt-get install docker-compose-plugin

# Verify V2 is installed
docker compose version
```

**Hidden files not visible after extracting?**

Press `Ctrl+H` in your file manager to show hidden files (`.claude/`, `.projexlight/`, `.mcp.json`).
